# Кейс-задача №5 — аналитический обзор
## Объект анализа: Кейс №4 «Интерактивный калькулятор» (HTML + CSS + JavaScript)

### 1) Функциональность
Приложение предоставляет два поля ввода и четыре операции: сложение, вычитание, умножение и деление. Результат выводится на странице в отдельном блоке. Реализована проверка корректности ввода: если введены не числа — выводится сообщение об ошибке. Также добавлена обработка деления на ноль.

Плюсы:
- Простая и понятная логика.
- Все базовые операции реализованы.
- Есть обработка ошибок (некорректный ввод, деление на 0).

Возможные улучшения:
- Добавить поддержку дробных чисел с запятой (например, 10,5).
- Добавить кнопку “Очистить” для сброса полей и результата.
- Добавить форматирование результата (например, округление до 2–4 знаков при необходимости).

---

### 2) Производительность
Нагрузка минимальная: выполняются простые арифметические операции и обновление одного DOM-элемента. Приложение работает мгновенно, без задержек.

Возможные улучшения:
- Объединить обработчики в одну универсальную функцию (уменьшить дублирование кода).
- Добавить дебаунс/валидацию при вводе (не обязательно, но возможно для расширения).

---

### 3) Удобство использования (юзабилити)
Интерфейс интуитивный: два поля ввода, кнопки операций и блок результата. Пользователь сразу видит, где вводить данные и где отображается итог.

Проблемы:
- Пользователь не получает визуальной подсказки, какое поле введено неверно.
- Нет подсказки по формату ввода.

Возможные улучшения:
- Подсвечивать поля красной рамкой при ошибке.
- Добавить текст-подсказку: “Можно вводить целые и дробные числа”.
- Добавить обработку Enter (например, запуск последней операции).

---

### 4) Безопасность
Приложение работает локально и не отправляет данные на сервер, поэтому риски минимальны. Вывод результата и ошибок осуществляется как текст, что снижает риск внедрения вредного HTML/скриптов.

Возможные улучшения:
- Всегда использовать вывод через textContent (не использовать innerHTML).
- При размещении онлайн — добавить заголовки безопасности на сервере (например, CSP).

---

### 5) Масштабируемость
Проект легко расширять: можно добавить новые операции (степень, корень, проценты), историю вычислений, переключение тем оформления.

Возможные улучшения:
- Вынести операции в структуру (объект/словарь) и делать вычисление по ключу операции.
- Добавить модульность: отдельные функции для валидации, вычисления и отображения.

---

### 6) Сопровождаемость
Код разделён на файлы (HTML/CSS/JS), структура простая, функции легко читаются. Это упрощает поддержку и поиск ошибок.

Возможные улучшения:
- Добавить комментарии к ключевым частям кода (валидация, обработчики).
- Добавить README с инструкцией запуска и возможностями (если ещё не добавлен).

---

### 7) Переносимость
Работает в любом современном браузере и не требует установки библиотек или сборки. Достаточно открыть index.html.

Возможные улучшения:
- Проверить отображение в нескольких браузерах (Chrome/Safari/Firefox).
- При необходимости — добавить адаптивность под мобильные устройства.

---

### 8) Качество кода
Код структурирован и понятен. Используются обработчики событий. Есть проверка ввода и обработка деления на ноль.

Возможные улучшения:
- Уменьшить дублирование (сделать одну функцию compute(operation)).
- Использовать более строгую проверку ввода (Number.isFinite).
- Добавить единый метод вывода ошибок.

---

### 9) Тестирование
Проведено ручное тестирование:
- Ввод корректных чисел (10 и 5) — операции выполняются правильно.
- Ввод дробных чисел (например, 2.5 и 1.5) — корректный результат.
- Ввод нечисловых значений (abc) — вывод ошибки.
- Деление на ноль — вывод ошибки.

Возможные улучшения:
- Добавить чек-лист тестов в README.
- При расширении проекта — добавить автотесты для функций вычисления.
